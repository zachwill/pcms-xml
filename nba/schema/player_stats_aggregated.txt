#### Notes
- `nba_id` and `team_id` refer to the player and the team they played for in that specific aggregation.
- **Season-long "Totals"**: If a player played for multiple teams during a season, the API provides a row with `teamId=0` (or `null`/`TOT` in some formats) to represent the combined season total. We preserve this by allowing `team_id` in the PK. Queries for "season-long stats" should filter for the specific `team_id` (e.g., 0 for combined, or a specific team ID for splits).
- `season_year` is the starting year of the season (e.g., 2023 for the 2023-24 season).
- `season_label` is the display string for the season (e.g., "2023-24").
- `season_type`: e.g., "Regular Season", "Playoffs", "Pre Season", "PlayIn".
- `per_mode`: The aggregation mode from the API (e.g., "Totals", "PerGame", "Per36", "Per100Possessions").
- `measure_type`: The set of stats (e.g., "Base", "Advanced", "Misc", "Scoring", "Usage").
- Most stat columns are `numeric` to accommodate both "Totals" (integers) and "PerGame" / "PerPossession" (floats) without loss of precision.
- `minutes`: Stored as `numeric` here because aggregated values are often decimal averages (e.g., 34.5 mins per game).
- `double_doubles`, `triple_doubles`: Counts of specific milestones.
- This table stores data from the `/api/stats/player` endpoint (Player Stats) which provides over 130 measures.
- We include common measures from "Base", "Advanced", and "Misc" types.
